# notes/

Daily personal notes from study sessions.

## Convention

One file per session, named `YYYY-MM-DD-topic.md`. Examples:
- `2026-06-01-cgroups-memory.md`
- `2026-06-02-multi-stage-builds.md`

## What to write

These are **your** notes, not Claude's. The act of writing in your own words is what locks in understanding.

Suggested template:

```markdown
# YYYY-MM-DD — Topic name

## What I learned today
- 3-5 bullet points in your own words

## What I built / tried hands-on
- The actual command / YAML / experiment you did

## What's still fuzzy
- 1-2 things you'd want to revisit

## Self-test answers
- Speak the answers to the self-test questions out loud, then write the gist
```

Don't overthink the format. The goal is recall when you read back later.
